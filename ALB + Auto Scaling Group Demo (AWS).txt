ALB + Auto Scaling Group Demo (AWS)

This project demonstrates a high-availability, auto-scaling web architecture on AWS using an Application Load Balancer (ALB), EC2 Auto Scaling Group (ASG), CloudWatch, and AWS Systems Manager (SSM).

The goal was to build something realistic, observable, and scalable â€” and to fully understand why each component exists and how they work together.

Architecture Overview

Services used:

Amazon EC2

Application Load Balancer (ALB)

Auto Scaling Group (ASG)

CloudWatch Metrics

IAM Roles

What this setup does:

Routes traffic through an ALB

Automatically scales EC2 instances based on CPU usage

Uses SSM instead of SSH for secure instance access

Verifies scaling behavior using real CPU stress tests

Key Takeaways

Auto Scaling responds to real load, not just configuration

ALB ensures zero-downtime traffic distribution

SSM is a cleaner, more secure alternative to SSH

CloudWatch metrics are essential for visibility and debugging
